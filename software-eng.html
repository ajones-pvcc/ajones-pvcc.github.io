Chapter 7: Software Engineering

Term Definition
software engineering a branch of computer science that uses engineering principles to develop computer programs and operating systems
integrated development environment (IDE) a software application that provides comprehensive facilities to computer programmers for software development
Association for Computing Machinery (ACM) international organization for computer science and information technology professionals
purpose of ACM it fosters research and communication in a broad range of computing areas
benefits of ACM membership you become a part of the dynamic changes that are transforming our world by helping to shape the future of computing
student ACM membership provides opportunities to students for networking, learning together and sharing their knowledge
Institute of Electrical and Electronics Engineers (IEEE) a professional association that develops, defines, and reviews electronics and computer science standards
purpose of IEEE development of standards for computer networking and its suite of services
benefits of IEEE membership offers access to technical innovation, cutting-edge information, and networking opportunities
student IEEE membership offers countless opportunities for students to connect with a global, and local, community of computing professionals to exchange ideas and learn from one other
Software Life Cycle: Development phase the process of planning, writing, and modifying software
Requirements analysis the efficient examination of requirements gathered, and clear requirement documentation as a precondition
Design the original plan and vision is elaborated into the basic structure of the software, including the system design, programming language, templates, platform to use, and application security measures
Implementation where the development team turns product specifications and business requirements into code that makes the product
Testing when a project goes through testing such as performance testing, functional testing, security testing, unit-testing, and usability testing
modular implementation a set of procedures performed to complete the design contained in the approved systems design document and to test, install, and begin to use the new or revised information system
Unifed Modeling Language (UML) a standardized modeling language consisting of an integrated set of diagrams, developed to help system and software developers for specifying, visualizing, constructing, and documenting the artifacts of software systems
use case a way to summarize details of a system and the users within that system
use case diagram used to depict the functionality of a system or a part of a system
Data relationships structural relationships between data items that deal directly with the architecture of a machine
one-to-one data relationship when one parent record or field has either zero or one child record only
one-to-many data relationship a principle of database design, which cannot be explicitly defined in the database structure
many-to-many data relationship a relationship between tables in a database when a parent row in one table contains several child rows in the second table
design pattern systematically names, motivates, and explains a general design that addresses a recurring design problem in object-oriented systems
Software quality assurance (SQA) a process that ensures that developed software meets and complies with defined or standardized quality specifications
Software testing the process of evaluating and verifying that a software product or application does what it is supposed to do
glass box testing a software testing technique that tests the internal workings of a software application
black box testing a software testing technique that focuses on the analysis of software functionality, versus internal system mechanisms
beta testing a phase of software testing in which a sampling of the intended audience tries the product out
alpha testing a type of software testing performed to identify bugs before releasing the product to real users or to the public
Documentation a way for engineers and programmers to describe their product and the process they used in creating it in formal writing
user documentation the documentation for a product or service provided to the end users
system documentation detailed information, in either written or computerized form, about a computer system
technical documentation the documentation that describes how a product or service operates
Intellectual property law computer code or software protected by law under either a copyright, trademark, trade secret, or software patent
software license a document that provides legally binding guidelines for the use and distribution of software
software disclaimer states the limitations of a softwares liability
best defenses against software liability claims to apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors
